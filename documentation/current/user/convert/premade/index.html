Prepackaged Converter DefinitionsÂ¶

GeoMesa Tools distributions contain several prepackaged converters for common, publically available data sources listed below. These are bundled with the command-line tools in the GeoMesa binary distributions in individual directories in conf/sfts, which are included by the conf/reference.conf file.

These prepackaged converters can also be obtained in a JAR that can be placed on the classpath of your project. This JAR may be obtained via Maven or SBT by requesting the geomesa-tools_2.11 artifact with the data classifier. For example, for Maven add the following to your POM:

<dependency>
    <groupId>org.locationtech.geomesa</groupId>
    <artifactId>geomesa-tools_2.11</artifactId>
    <version>${geomesa.version}</version>
    <classifier>data</classifier>
</dependency>

Note

The examples in the sections below are often specific to the GeoMesa Accumulo distribution, but the general principle is the same for each distribution. Only the home variable (e.g. $GEOMESA_ACCUMULO_HOME) and command-line tool name will differ depending on GeoMesa distribution.

  • Global Database of Events, Language, and Tone (GDELT)
  • GeoLife GPS Trajectory Data
  • GeoNames
  • Global Terrorism Database (GTD)
  • NYC Taxi
  • OpenStreetMap GPX Data
  • T-Drive Trajectory Data
  • Twitter
